Ingeniería del Modelo: Predicción de Malnutrición Infantil

1. Selección de Algoritmos

Se desarrollo la Ingeniería del Modelo utilizando Árbol de Decisión como algoritmo principal.

Selección del Algoritmo: Árbol de Decisión
Justificación:
  • Es fácil de interpretar y visualizar.
  • Funciona bien con variables categóricas y numéricas.
  • Tiene buen rendimiento en datasets pequeños o medianos como el tuyo (500 registros).
Datos utilizados:
  • Variables predictoras: peso, altura, edad, sexo, lugar.
  • Variable objetivo: estado nutricional (Bajo peso, Normal, Sobrepeso), calculado mediante el IMC.

2. Entrenamiento

Objetivo:

Aplicar los datos transformados al algoritmo seleccionado.

Acción sugerida:
  • Divide el dataset en entrenamiento y prueba.
  • Usa las variables: peso, altura, edad, sexo, lugar para predecir el estado nutricional.
  • Aplica el algoritmo y ajusta para que aprenda la relación entre las variables y la etiqueta nutricional.
Ajuste de Hiperparámetros
Enfoque aplicado:

Se utilizó un dataset existente (posterior al proceso de limpieza y transformación) y se solicitó a ChatGPT ajustar los hiperparámetros del modelo de Árbol de Decisión con base en las variables disponibles (peso, altura, edad, sexo, lugar) para mejorar su rendimiento en la predicción del estado nutricional.

3. Hiperparámetros ajustados:

  • max_depth: se limitó para evitar sobreajuste y mantener interpretabilidad.
  • min_samples_split: se configuró para controlar la mínima cantidad de muestras necesarias para dividir un nodo.
  • criterion: se probó tanto "gini" como "entropy" para evaluar la mejor opción de división.
Método utilizado:

Se realizó un proceso de prueba y error guiado por ChatGPT para encontrar la combinación óptima de hiperparámetros que ofreciera el mejor balance entre precisión y simplicidad del árbol.

Resultado:

Se obtuvo un árbol más eficiente, con mejor generalización y comprensión lógica del problema, ideal para su uso en contextos sociales y de salud pública.

4. Validación

Objetivo:

Evaluar el desempeño del modelo de Árbol de Decisión usando datos que no fueron utilizados durante el entrenamiento, para asegurar que generaliza correctamente a nuevos casos.

Procedimiento aplicado:
  1. División del conjunto de datos:
    • El dataset ajustado se dividió en:
      • 80% para entrenamiento
      • 20% para prueba
    • Esto permite evaluar el rendimiento real sin introducir sesgo de sobreentrenamiento.
  2. Métricas utilizadas:
    • Precisión (Accuracy): Proporción de clasificaciones correctas.
    • Matriz de Confusión: Para analizar los errores específicos entre clases (Bajo peso, Normal, Sobrepeso).
  3. Resultados esperados:
    • Alta precisión en la clasificación del estado nutricional.
    • Buen comportamiento en todas las clases, especialmente en la clase “Bajo peso” por su relevancia social.
